# PMS Team Project

팀 프로젝트용 저장소입니다.

복잡한 규칙보다는, **각자 맡은 기능을 브랜치에서 작업하고 PR로 합치는 방식**으로 운영합니다.

## 폴더 구성
- `PMS 로그인 api (...)` : 로그인/인증 관련 API 작업
- `PMS 맴버 일정(...)` : 멤버/일정 관련 기능 작업

## 브랜치 사용 (간단 버전)
- `main` : 정리된 코드
- `feature/login-api` : 로그인 API 작업 브랜치
- `feature/member-schedule` : 멤버/일정 기능 작업 브랜치

## 작업 흐름
1. 내 작업 브랜치에서 개발
2. 커밋 후 push
3. PR로 리뷰 후 `main`에 머지

## 버전 태그
- `v0.1.0` : 초기 업로드
- `v0.1.1` : `.gitignore` 추가 + 빌드 산출물 정리
